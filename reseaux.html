NIVEAU 1
Switch de base
Niveau 1 · Premier contact

Accès à l'équipement & nommage

Tu viens de brancher ton câble console sur un switch Cisco. Par défaut, tu arrives en mode utilisateur limité. La première chose à faire est de passer en mode privilégié, puis en mode configuration pour pouvoir modifier l'équipement. Ces deux commandes sont le point de départ de toute configuration Cisco.
accès-et-nommage.ios
Switch / Routeur
!! Passe du mode utilisateur au mode privilégié enable !! Entre dans le mode configuration globale configure terminal !! ou "conf t" en abrégé !! Donne un nom à l'équipement hostname SW01 !! Remplace SW01 par ton nom souhaité !! Désactive la résolution DNS (évite les délais inutiles) no ip domain-lookup
enable
Mode privilégié — Donne accès aux commandes de diagnostic et de configuration. Le prompt passe de > à #.
conf t
Mode configuration globale — Permet de modifier la configuration. Le prompt devient (config)#.
hostname SW01
Nomme l'équipement. Apparaît dans le prompt et dans les logs réseau. Convention : SW pour switch, R pour routeur + numéro.
no ip domain-lookup
Empêche le switch de chercher un serveur DNS à chaque faute de frappe. Sans cette commande, une typo provoque 30 secondes d'attente.
Niveau 1 · Segmentation réseau

Création de VLANs

Un VLAN (Virtual LAN) permet de segmenter logiquement un switch en plusieurs réseaux isolés, sans câblage supplémentaire. Exemple : VLAN 101 pour les postes clients, VLAN 102 pour les serveurs, VLAN 99 pour la gestion. Les machines de VLANs différents ne communiquent pas sans un routeur ou switch de niveau 3.
creation-vlans.ios
Switch
!! Création du VLAN 101 et attribution d'un nom vlan 101 name CLIENT !! On peut enchaîner plusieurs VLANs vlan 102 name SERVEUR vlan 103 name INVITE vlan 99 name GESTION !! Revenir au mode configuration globale exit
vlan 101
Crée le VLAN numéro 101. Le numéro identifie le VLAN sur tout le réseau. Plage standard : 1–1005. Plage étendue : 1006–4094.
name CLIENT
Donne un nom lisible au VLAN. Facilite la gestion. Le nom n'a aucun impact technique.
exit
Remonte d'un niveau dans la hiérarchie de configuration. Ici, quitte le mode VLAN pour revenir en configuration globale.
Niveau 1 · Affectation des ports

Interfaces Access et Trunk

Deux types de ports sur un switch : Access (port branché à un PC ou serveur, appartient à un seul VLAN) et Trunk (port entre deux switches ou vers un routeur, transporte tous les VLANs simultanément avec le tag 802.1Q).
ports-access.ios
Affecter des ports à un VLAN
!! Sélectionner une plage d'interfaces (fa0/1 à fa0/15) interface range fa0/1 - 15 switchport mode access !! Mode access = 1 seul VLAN switchport access vlan 101 !! Affecte au VLAN 101 !! Autre plage pour un autre VLAN interface range fa0/16 - 20 switchport mode access switchport access vlan 102
ports-trunk.ios
Trunk entre switches
!! Ports reliant ce switch au switch cœur (ou au routeur) interface range fa0/21 - 24 switchport mode trunk !! Mode trunk = tous les VLANs passent switchport trunk allowed vlan all !! Autorise tous les VLANs !! Sur switch Catalyst 3560 (niveau 3), encapsulation obligatoire avant trunk switchport trunk encapsulation dot1q switchport mode trunk switchport trunk allowed vlan all
interface range fa0/1 - 15
Sélectionne plusieurs ports en une seule commande. Évite de répéter les mêmes commandes port par port.
switchport mode access
Fixe le port en mode access. Le port n'appartient qu'à un seul VLAN. À utiliser sur les ports connectés aux équipements terminaux (PC, imprimante, serveur).
switchport mode trunk
Active le trunk. Le port transporte plusieurs VLANs en ajoutant un tag 802.1Q à chaque trame. À utiliser entre switches ou vers un routeur.
encapsulation dot1q
Uniquement sur switch L3 (3560, 3750). Choisit le protocole d'encapsulation 802.1Q pour le trunk. Obligatoire sur ces modèles avant switchport mode trunk.

Niveau 2 · Synchronisation

VTP — Propagation automatique des VLANs

VTP (VLAN Trunking Protocol) permet de centraliser la gestion des VLANs sur un seul switch "serveur" qui propage automatiquement les VLANs vers tous les switches "clients". Prérequis absolu : les liaisons entre switches doivent être en trunk.
vtp-server.ios
Switch cœur — serveur VTP
vtp mode server !! Ce switch distribue les VLANs vtp domain tssr !! Nom du domaine VTP (identique sur tous) vtp password cisco !! Mot de passe VTP (identique sur tous) !! Ensuite créer les VLANs ici — ils seront propagés automatiquement vlan 100 name WIFI vlan 101 name SERVEUR
vtp-client.ios
Switches secondaires — clients VTP
vtp mode client !! Reçoit les VLANs, ne peut pas en créer vtp domain tssr !! Doit être identique au serveur vtp password cisco !! Doit être identique au serveur
⚠️
En mode client VTP, il est impossible de créer des VLANs localement. Tous les VLANs doivent être créés sur le serveur VTP. Le domaine et le mot de passe doivent être strictement identiques sur tous les switches.
Niveau 2 · Sécurité d'accès

Accès distant sécurisé (Telnet/SSH)

Pour administrer un switch à distance via le réseau (sans câble console), il faut configurer les lignes VTY et définir un mot de passe. Cette configuration s'applique à tous les switches du réseau.
acces-distant.ios
À appliquer sur tous les switches
!! Définir un mot de passe pour le mode privilégié (enable) enable secret cisco !! Configurer les 16 lignes VTY (accès Telnet/SSH simultanés) line vty 0 15 password distance !! Mot de passe de connexion login !! Active la demande du mot de passe
enable secret cisco
Définit le mot de passe pour entrer en mode privilégié (#). Secret = stocké chiffré dans la config (préférable à enable password qui est en clair).
line vty 0 15
Sélectionne les 16 lignes virtuelles (0 à 15). Chaque ligne = une session Telnet/SSH simultanée possible.
login
Oblige la saisie du mot de passe lors de la connexion. Sans cette commande, le mot de passe défini est ignoré.
Niveau 2 · Routage inter-VLAN

Routage inter-VLAN sur switch L3 (3560)

Un switch de niveau 3 (Cisco 3560, 3750) peut faire du routage sans routeur externe. On crée des interfaces VLAN virtuelles qui servent de passerelle pour chaque VLAN, puis on active le routage IP. C'est la solution la plus performante en entreprise.
intervlan-l3.ios
Switch Catalyst 3560 — switch cœur
!! 1. Créer les VLANs vlan 101 name CLIENT vlan 102 name SERVEUR vlan 99 name GESTION exit !! 2. Passer les ports uplink en trunk (dot1q obligatoire sur 3560) interface range fa0/1-20 switchport trunk encapsulation dot1q switchport mode trunk switchport trunk allowed vlan all exit !! 3. Créer les interfaces VLAN (passerelles) interface vlan 101 ip address 192.168.101.254 255.255.255.0 !! GW du VLAN 101 interface vlan 102 ip address 192.168.102.254 255.255.255.0 interface vlan 99 ip address 192.168.99.254 255.255.255.0 exit !! 4. Activer le routage IP inter-VLAN ip routing !! INDISPENSABLE — sans ça les VLANs restent isolés
interface vlan 101
Crée une interface logique (SVI) pour le VLAN 101. L'adresse IP assignée sera la passerelle par défaut de tous les équipements du VLAN 101.
ip routing
Active le routage inter-VLAN. Sans cette commande, le switch connaît les VLANs mais ne route pas les paquets entre eux. C'est la commande clé du switch L3.
Niveau 2 · Alternative avec routeur

Routage inter-VLAN via routeur (Router-on-a-Stick)

Quand on n'a pas de switch L3, on relie un routeur à un port trunk du switch, et on crée une sous-interface par VLAN sur le routeur. Chaque sous-interface reçoit une IP de passerelle. Le câble physique est unique mais transporte tous les VLANs (d'où le nom "router-on-a-stick").
router-on-stick.ios
Routeur 1841 — 1 sous-interface par VLAN
!! 1. Allumer l'interface physique (sans adresse IP) interface fa0/0 no shutdown !! 2. Créer une sous-interface par VLAN !! Format : fa0/0.NUMERO_VLAN interface fa0/0.99 encapsulation dot1q 99 !! Associe la sous-interface au VLAN 99 ip address 192.168.99.254 255.255.255.0 interface fa0/0.101 encapsulation dot1q 101 ip address 192.168.101.254 255.255.255.0 interface fa0/0.102 encapsulation dot1q 102 ip address 192.168.102.254 255.255.255.0
interface fa0/0.101
Crée une sous-interface logique sur fa0/0. La convention est d'utiliser le numéro de VLAN comme suffixe. Le routeur peut en gérer des dizaines sur une seule interface physique.
encapsulation dot1q 101
Indique au routeur que cette sous-interface doit traiter les trames taguées VLAN 101. Le numéro doit correspondre au VLAN déclaré sur le switch.
Niveau 2 · Gestion réseau

Adresse IP de gestion & Gateway sur switch L2

Un switch de niveau 2 n'a pas d'interface IP par VLAN par défaut. Pour l'administrer à distance (Telnet/SSH), on lui donne une IP sur l'interface VLAN de gestion (souvent VLAN 99). Comme il ne route pas, il a besoin d'une passerelle par défaut pour répondre aux requêtes venant d'autres VLANs.
gestion-switch-l2.ios
Switch 2960 — IP de gestion
!! Donner une IP au switch sur le VLAN de gestion interface vlan 99 ip address 192.168.99.253 255.255.255.0 exit !! Ajouter la passerelle par défaut (adresse IP du switch L3 sur ce VLAN) ip default-gateway 192.168.99.254
💡
Chaque switch L2 doit avoir une IP différente sur le VLAN de gestion. Par convention, le switch cœur prend .254, le switch gauche .253, le switch droit .252, etc.
Niveau 2 · Maintenance

Réinitialisation complète d'un switch Cisco

Quand on veut repartir de zéro sur un switch (ou avant de le reconfigurer), il faut supprimer la configuration de démarrage ET la base de données VLANs, qui est stockée séparément dans flash.
reset-switch.ios
Mode privilégié (#)
!! 1. Supprimer la configuration de démarrage write erase !! 2. Supprimer la base de données VLANs (stockée séparément) delete flash:vlan.dat !! 3. Redémarrer le switch reload
⚠️
Si on oublie delete flash:vlan.dat, les VLANs persistent au redémarrage même si la config est effacée. Ces deux commandes sont indissociables pour une réinitialisation complète.

Niveau 3 · Routage

Routage statique

Le routage statique consiste à indiquer manuellement à chaque routeur par où envoyer les paquets. C'est l'administrateur qui contrôle totalement le chemin. Distance administrative = 1 (très prioritaire). La route par défaut (0.0.0.0 0.0.0.0) correspond à "tout ce que je ne connais pas, envoie là".
routage-statique.ios
Syntaxe : ip route DESTINATION MASQUE PROCHAIN_SAUT
!! Route par défaut = tout envoyer vers 100.15.1.30 (souvent vers FAI) ip route 0.0.0.0 0.0.0.0 100.15.1.30 !! Route spécifique vers le réseau 192.168.2.0/24 via 10.10.10.6 ip route 192.168.2.0 255.255.255.0 10.10.10.6 !! Route vers un réseau /16 via une passerelle ip route 179.58.0.0 255.255.0.0 64.0.0.1
ip route 0.0.0.0 0.0.0.0 X.X.X.X
Route par défaut. "Si je ne sais pas où envoyer ce paquet, je l'envoie vers X.X.X.X." Équivalent de la gateway par défaut d'un PC. Utilisée pour sortir vers Internet.
ip route DEST MASQUE NEXT-HOP
Syntaxe complète. DEST = réseau destination, MASQUE = masque de ce réseau, NEXT-HOP = IP du prochain routeur qui sait y aller.
Niveau 3 · Routage dynamique

Routage dynamique RIP v2

RIP (Routing Information Protocol) est un protocole de routage dynamique : les routeurs échangent automatiquement leurs tables de routage. Distance administrative = 120. On déclare uniquement les réseaux directement connectés à chaque routeur. No auto-summary est indispensable pour supporter les sous-réseaux.
rip-v2.ios
Même structure sur chaque routeur
router rip version 2 !! RIPv2 supporte VLSM et les sous-réseaux no auto-summary !! INDISPENSABLE — désactive la summarisation automatique !! Déclarer UNIQUEMENT les réseaux directement connectés à ce routeur network 192.168.1.0 !! Réseau côté LAN network 10.10.10.0 !! Réseau point-à-point vers autre routeur
📌
Avec RIP, on déclare les réseaux connectés directement au routeur, pas les réseaux distants. Le protocole s'occupe de propager l'info aux voisins automatiquement.
Niveau 3 · Routage dynamique avancé

Routage EIGRP

EIGRP est un protocole de routage dynamique Cisco plus évolué que RIP. Distance administrative = 90 (plus prioritaire que RIP). Il supporte les masques variables (VLSM) et converge rapidement. Le numéro de système autonome (AS) doit être identique sur tous les routeurs du même domaine EIGRP.
eigrp.ios
AS 92 sur tous les routeurs du domaine
router eigrp 92 !! 92 = numéro du système autonome (AS) — identique partout no auto-summary !! Réseaux classful (classe entière) — pas besoin de wildcard network 192.168.1.0 !! Réseaux avec masque différent de la classe — ajouter le wildcard mask network 100.15.1.28 0.0.0.3 !! Réseau /30 → wildcard = 0.0.0.3 network 64.0.0.0 0.0.0.3 !! Réseau /30
router eigrp 92
Lance le processus EIGRP avec l'AS numéro 92. Tous les routeurs d'un même réseau EIGRP doivent utiliser le même numéro AS.
network X.X.X.X 0.0.0.3
Le wildcard mask est le masque inversé. Pour /30 (255.255.255.252), le wildcard est 0.0.0.3. Nécessaire uniquement si le masque est différent de la classe naturelle.

Niveau 3 · Interfaces WAN

Configuration d'interfaces (FA et Série)

Les interfaces FastEthernet (fa0/0) connectent un routeur au LAN ou à un autre équipement réseau. Les interfaces Série (se0/0/0) connectent deux routeurs via un lien WAN simulé. Sur le lien série, l'un des deux routeurs doit fournir la clock rate (côté DCE).
interfaces-routeur.ios
FA + Série avec clock rate
!! Interface FastEthernet vers le LAN interface fa0/0 ip address 192.168.1.254 255.255.255.0 no shutdown !! INDISPENSABLE — les interfaces sont off par défaut !! Interface Série WAN — côté qui fournit l'horloge (DCE) interface se0/0/0 ip address 10.10.10.5 255.255.255.252 clock rate 4000000 !! Seulement sur le côté DCE du câble série no shutdown
💡
no shutdown est obligatoire sur chaque interface. Sans lui, l'interface reste éteinte même si l'IP est configurée. C'est une des erreurs les plus fréquentes en TP.
Niveau 3 · NAT

NAT — Déclaration inside / outside

NAT (Network Address Translation) traduit les adresses IP privées en adresses publiques pour sortir sur Internet. Première étape obligatoire : déclarer sur chaque interface si elle est côté réseau interne (inside) ou côté Internet (outside).
nat-inside-outside.ios
Toujours faire ça en premier
!! Interface côté réseau privé (LAN) interface fa0/0 ip nat inside !! Réseau privé = inside exit !! Autre interface LAN (si plusieurs réseaux privés) interface fa0/1 ip nat inside exit !! Interface côté Internet / FAI interface se0/0/0 ip nat outside !! Internet = outside
Niveau 3 · NAT Statique

NAT Statique — 1 IP privée = 1 IP publique

Le NAT statique associe de façon permanente une adresse IP privée à une adresse IP publique. Utilisé pour exposer un serveur interne sur Internet. Une seule machine peut utiliser cette adresse publique. Exemple : serveur web 192.168.1.150 accessible via l'IP publique 201.49.10.17.
nat-statique.ios
Après avoir déclaré inside/outside
!! Associer l'IP privée 192.168.1.150 à l'IP publique 201.49.10.17 ip nat inside source static 192.168.1.150 201.49.10.17
Niveau 3 · NAT Dynamique

NAT Dynamique — Pool d'adresses publiques

Le NAT dynamique permet à plusieurs machines privées de partager un pool d'adresses publiques. Attention : les machines ne peuvent pas sortir simultanément car chaque adresse publique n'est utilisée que par une machine à la fois.
nat-dynamique.ios
3 étapes obligatoires
!! Étape D.1 — Créer le pool d'adresses publiques ip nat pool POOL-NAT 201.49.10.18 201.49.10.20 netmask 255.255.255.252 !! Étape D.2 — ACL pour définir qui peut utiliser le NAT !! (Exclure les machines en NAT statique si besoin) access-list 22 deny host 192.168.1.150 !! Exclure le serveur en NAT statique access-list 22 permit 192.168.1.0 0.0.0.255 !! Autoriser tout le reste du réseau !! Étape D.3 — Appliquer le NAT dynamique ip nat inside source list 22 pool POOL-NAT
Niveau 3 · PAT

PAT / Surcharge NAT — Toutes les machines sur 1 IP publique

Le PAT (Port Address Translation) est la forme la plus utilisée du NAT. Toutes les machines internes partagent la même IP publique, différenciées par leur numéro de port source. C'est ainsi que fonctionne votre box internet à la maison. Le mot-clé overload active le PAT.
pat-overload.ios
Le plus utilisé en production
!! Étape P.1 — ACL pour définir les machines autorisées access-list 33 permit 192.168.0.0 0.0.0.255 !! Étape P.2 — Appliquer le PAT sur l'interface outside !! Le mot-clé OVERLOAD active le partage de l'IP publique par les ports ip nat inside source list 33 interface se0/0/0 overload
Niveau 3 · NAT avancé

Redirection de port (Port Forwarding)

La redirection de port permet d'exposer un service interne sur Internet via une IP publique. Exemple : les requêtes HTTP (port 80) arrivant sur l'IP publique du routeur sont redirigées vers un serveur web interne.
port-forwarding.ios
NAT statique avec port TCP
!! Rediriger HTTP (80) de l'IP publique 7.0.0.100 vers le serveur interne 10.0.0.12 ip nat inside source static tcp 10.0.0.12 80 7.0.0.100 80 !! Syntaxe : ip nat inside source static tcp IP_PRIVÉE PORT_PRIVÉ IP_PUBLIQUE PORT_PUBLIC !! Exemple HTTPS aussi : ip nat inside source static tcp 192.168.20.99 443 84.96.154.6 443

Niveau 4 · Sécurité réseau

ACL Standard — Filtrage par IP source

Une ACL Standard filtre le trafic uniquement par adresse IP source. Numéros 1 à 99. Règle d'or : placer l'ACL standard le plus près de la destination. Le routeur analyse les règles dans l'ordre séquentiel. Si aucune règle ne correspond → refus implicite (tout est bloqué par défaut).
acl-standard.ios
2 étapes : créer la règle + appliquer sur interface
!! ÉTAPE 1 : Créer les règles ACL 92 !! Bloquer le réseau 172.16.1.0/24 access-list 92 deny 172.16.1.0 0.0.0.255 !! Autoriser tout le reste (sans cette ligne = tout bloqué) access-list 92 permit any !! ÉTAPE 2 : Appliquer l'ACL sur une interface interface fa0/0 ip access-group 92 out !! out = trafic sortant de cette interface !! Pour bloquer une seule machine : access-list 72 deny host 192.168.0.1 access-list 72 permit any
SyntaxeSignification
deny 10.0.0.0 0.255.255.255Bloquer tout le réseau 10.0.0.0/8
permit host 192.168.1.1Autoriser uniquement cette machine
deny host 10.0.0.1Bloquer uniquement cette machine
permit anyAutoriser tout le reste
ip access-group X inFiltrer le trafic entrant sur l'interface
ip access-group X outFiltrer le trafic sortant de l'interface
Niveau 4 · Sécurité avancée

ACL Étendue — Filtrage IP source + destination + port

Une ACL étendue filtre par IP source, IP destination, protocole et port. Numéros 100 à 199. Règle d'or : placer l'ACL étendue le plus près de la source pour éviter de laisser circuler le trafic inutilement.
acl-etendue.ios
Numéros 100-199
!! Syntaxe : access-list NUM ACTION PROTOCOLE SRC WILDCARD DST WILDCARD [PORT] !! Bloquer le trafic HTTP du réseau 172.16.1.0 vers 192.168.1.32/27 access-list 110 deny tcp 172.16.1.0 0.0.0.255 192.168.1.32 0.0.0.31 eq 80 access-list 110 permit ip any any !! Autoriser le reste !! Appliquer sur l'interface la plus proche de la source interface fa0/0 ip access-group 110 in
Niveau 4 · Services réseau

DHCP sur routeur Cisco

Un routeur Cisco peut distribuer automatiquement des adresses IP aux équipements du réseau, sans serveur DHCP dédié. On crée un pool DHCP avec le réseau, la passerelle et le DNS, puis on exclut les adresses réservées aux équipements fixes.
dhcp-routeur.ios
Service DHCP natif Cisco
!! Créer le pool DHCP nommé CLIENT_LAN ip dhcp pool CLIENT_LAN network 192.168.0.0 255.255.255.0 !! Étendue d'adresses distribuées dns-server 8.8.8.8 !! DNS Google (ou autre) default-router 192.168.0.1 !! Passerelle par défaut pour les clients exit !! Exclure des adresses du pool (réservées aux équipements fixes) ip dhcp excluded-address 192.168.0.240 192.168.0.250 !! Pool pour téléphones IP (VLAN voix) ip dhcp pool ETENDUE_TELEPHONE network 192.168.20.0 255.255.255.0 default-router 192.168.20.254 option 150 ip 192.168.20.254 !! Option 150 = adresse du serveur TFTP pour les téléphones Cisco
Niveau 4 · DHCP relay

DHCP Relay — ip helper-address

Les requêtes DHCP fonctionnent en broadcast, donc elles sont bloquées par les routeurs. Pour permettre aux clients d'obtenir une IP depuis un serveur DHCP situé dans un autre réseau, on configure le routeur en "relay agent" : il transforme le broadcast en unicast vers le serveur DHCP.
dhcp-relay.ios
Sur l'interface côté clients
!! Sur l'interface FA du routeur côté réseau clients interface fa0/1 !! Rediriger les broadcasts DHCP vers le serveur 192.168.1.2 ip helper-address 192.168.1.2
📌
Le processus DHCP se déroule en 4 étapes : Discover (broadcast) → Offer → Request → Ack. Le relay intercepte le Discover et l'envoie en unicast au serveur DHCP.
Niveau 4 · VoIP

VLAN Voix & Configuration téléphonie IP

Un VLAN voix (switchport voice vlan) permet au switch de prioriser le trafic téléphonique sur les mêmes ports que les données. Le routeur joue le rôle de serveur de téléphonie (Call Manager Express) en plus du routage inter-VLAN.
vlan-voix-switch.ios
Switch — VLAN data + voix sur même port
vlan 10 name data vlan 20 name voice !! Port trunk vers le routeur interface g0/1 switchport mode trunk switchport trunk allowed vlan all exit !! Ports access avec VLAN voix en parallèle interface range fa0/1 - 24 switchport mode access switchport access vlan 10 !! VLAN data pour le PC switchport voice vlan 20 !! VLAN voix pour le téléphone IP branché au même port
telephonie-routeur.ios
Routeur — Call Manager Express
telephony-service max-ephones 42 !! Nombre max de téléphones max-dn 42 !! Nombre max de numéros internes ip source-address 192.168.20.254 port 2000 !! IP du serveur téléphonie auto assign 1 to 42 !! Niveau 5 · IPv6

IPv6 — Bases, notation et types d'adresses

IPv6 utilise des adresses de 128 bits notées en hexadécimal. Les règles de simplification RFC 5652 permettent de réduire l'écriture. Le masque s'exprime uniquement en CIDR (/64 le plus souvent).
Type d'adresse IPv6Équivalent IPv4Usage
::1127.0.0.1Loopback
::0.0.0.0Toutes les adresses
2000::/3Adresses publiquesRoutable sur Internet
FE80::/10Adresses privéesNon routable — lien local uniquement
FC00::/7—Unique local (usage interne uniquement)
FF00::/8224.0.0.0/4Multicast
📌
Règles de simplification RFC 5652 : les zéros en tête sont facultatifs · les adresses s'écrivent en minuscules · un seul groupe de blocs 0000 consécutifs peut être remplacé par :: · si plusieurs groupes égaux, on remplace celui de gauche.
Niveau 5 · IPv6

Routage statique IPv6

La configuration IPv6 sur routeur nécessite d'activer le routage unicast IPv6 avant tout. On assigne ensuite une adresse globale à chaque interface, et une adresse link-local (FE80). La route statique se configure avec ipv6 route.
routage-ipv6.ios
Routeur CAFE — exemple complet
!! 1. Activer le routage IPv6 (obligatoire) ipv6 unicast-routing !! 2. HSRP — Redondance de routeur (Cisco)

HSRP (Hot Standby Router Protocol) crée une adresse IP virtuelle partagée entre deux routeurs. Les clients pointent vers cette IP virtuelle comme passerelle. Si le routeur principal tombe, le secondaire prend automatiquement le relais. Nécessite 3 IPs pour 2 routeurs.
hsrp.ios
BOX01 = priorité haute (actif) / BOX02 = standby
!! Sur BOX01 (routeur principal — priorité 105 > 100) interface fa0/0 standby 15 ip 192.168.100.100 !! IP virtuelle commune aux 2 routeurs standby 15 priority 105 !! Priorité haute = routeur actif standby 15 preempt !! Reprend le rôle actif si la prio redevient la plus haute standby 15 track fa0/1 !! Baisse la prio de 10 si fa0/1 tombe !! Sur BOX02 (routeur secondaire — priorité 100) interface fa0/0 standby 15 ip 192.168.100.100 !! Même IP virtuelle standby 15 priority 100 standby 15 preempt standby 15 track fa0/1

VRRP & GLBP — Alternatives à HSRP

VRRP est la version standard (multi-constructeurs) de HSRP. Il n'a besoin que de 2 IPs au lieu de 3. GLBP va plus loin en permettant la répartition de charge entre les routeurs (load balancing), avec un pourcentage configurable pour chaque routeur.
vrrp-glbp.ios
VRRP = standard / GLBP = load balancing
!! VRRP — syntaxe identique à HSRP avec le mot-clé vrrp !! BOX01 interface fa0/0 vrrp 15 ip 192.168.100.101 vrrp 15 priority 105 vrrp 15 preempt !! GLBP — load balancing entre les 2 routeurs !! BOX01 gère 25% du trafic, BOX02 gère 75% interface fa0/0 glbp 15 ip 192.168.100.100 glbp 15 priority 105 glbp 15 preempt glbp 15 load-balancing 25 !! Ce routeur prend 25% du trafic

Ports TCP/UDP — À connaître par cœur

Les numéros de port associent un service réseau à une application. Ports bien connus : 1 à 1023. Ports dynamiques (navigateurs, apps) : 1024 à 65535. Ces ports sont utilisés dans les ACL étendues et les redirections de port.

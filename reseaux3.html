PC / Hôte
Couche utilisateur — Configuration OS
Couche utilisateur
Le PC n'utilise pas le CLI Cisco. La configuration réseau se fait via le système d'exploitation (Windows). C'est la base de tout scénario : sans configuration IP correcte sur le poste, rien ne fonctionnera en amont.
Configuration IP (Windows)
ipconfig /all
Affiche toute la configuration IP : adresse, masque, passerelle, serveur DNS, DHCP activé ou non, adresse MAC. Premier réflexe de diagnostic.
ipconfig /release
Libère le bail DHCP actuel. L'interface perd son IP. À utiliser avant /renew pour forcer un renouvellement.
ipconfig /renew
Demande un nouveau bail DHCP. Utile après une modification du serveur DHCP ou pour diagnostiquer une attribution d'IP défaillante.
netsh interface ip set address "Ethernet" static 192.168.1.10 255.255.255.0 192.168.1.254
Configure une IP fixe en ligne de commande. Équivalent à remplir la case "Utiliser l'adresse IP suivante" dans les propriétés de la carte réseau.
netsh interface ip set dns "Ethernet" static 192.168.1.1
Configure un serveur DNS fixe. Sans DNS correct, la résolution de noms échoue même si le réseau IP fonctionne parfaitement.
Tests de connectivité
ping <IP_passerelle>
Teste l'accessibilité de la passerelle par défaut. C'est le premier test à faire. Si la passerelle ne répond pas, inutile de chercher plus loin : le problème est local (câblage, VLAN, IP mal configurée).
ping <IP_distante>
Teste la connectivité vers un hôte distant. Si la passerelle répond mais pas la cible, le problème est au niveau du routage ou d'un firewall intermédiaire.
tracert <IP>
Trace le chemin paquet par paquet, saut par saut. Identifie précisément à quel routeur ou équipement la communication échoue.
nslookup <nom_domaine>
Vérifie la résolution DNS. Permet de savoir si le problème est IP (réseau) ou DNS (résolution de nom). Indispensable pour différencier les deux.
arp -a
Affiche le cache ARP local : correspondances IP ↔ MAC. Utile pour vérifier qu'un voisin est bien connu en couche 2, ou détecter un conflit d'adresses.

Switch Niveau 2
Couche 2 — Commutation MAC, VLANs
Couche 2
Le switch L2 opère uniquement en couche 2 (adresses MAC). Il segmente le trafic par VLAN mais ne route pas entre eux. Deux machines sur des VLANs différents ne peuvent pas communiquer sans passer par un équipement de couche 3 (routeur ou switch L3). On entre ici dans le CLI Cisco.
Accès et configuration de base
enable
Passe en mode privilégié (prompt Switch#). Obligatoire avant toute commande de vérification ou de configuration.
configure terminal
Passe en mode configuration globale (prompt Switch(config)#). Toutes les commandes de paramétrage s'effectuent dans ce mode.
hostname SW-NOM
Renomme l'équipement. Le nom apparaît dans le prompt CLI et dans les logs réseau. Indispensable sur un réseau multi-switches pour identifier rapidement qui on configure.
enable secret <mot_de_passe>
Définit le mot de passe d'accès au mode privilégié, chiffré MD5. Toujours préférer enable secret à enable password qui stocke le mot de passe en clair.
Gestion des VLANs
vlan <id>
Crée un VLAN et entre dans sa configuration. Un VLAN doit être créé sur chaque switch du domaine pour pouvoir y transiter, même sur les trunks.
name <nom_vlan>
Donne un nom lisible au VLAN. Facilite l'identification dans show vlan brief et dans la documentation.
interface fa0/X
Sélectionne un port spécifique pour le configurer. Peut aussi s'utiliser avec interface range fa0/1-12 pour sélectionner plusieurs ports en une seule commande.
switchport mode access
Configure le port en mode access : il n'appartient qu'à un seul VLAN et les trames ne sont pas taguées. C'est le mode pour les ports connectés aux postes clients, imprimantes, etc.
switchport access vlan <id>
Attribue le port à un VLAN spécifique. Les trames reçues sur ce port seront associées à ce VLAN dans la table CAM du switch.
switchport mode trunk
Configure le port en mode trunk : il transporte plusieurs VLANs simultanément avec des trames taguées (802.1Q). Utilisé sur les liens inter-switches et vers les routeurs/switches L3.
switchport trunk allowed vlan <liste>
Restreint les VLANs autorisés sur un trunk (ex: 1,101,102,103 ou all). Bonne pratique : n'autoriser que les VLANs nécessaires pour limiter les broadcast inutiles.
show vlan brief
Affiche la liste des VLANs et les ports assignés à chacun. Commande de vérification indispensable après la configuration des VLANs.
show interfaces trunk
Liste les ports en mode trunk et les VLANs autorisés/actifs sur chacun. Permet de vérifier que les bons VLANs transitent bien sur chaque lien.
Interface de gestion (VLAN de management)
interface vlan <id>
Crée l'interface virtuelle de gestion du switch (SVI — Switched Virtual Interface). C'est la seule IP que possède un switch L2 : elle sert uniquement à l'administration (SSH, Telnet, ping de test).
ip address <IP> <masque>
Attribue une adresse IP à l'interface de gestion. Cette IP doit être dans le même réseau que la passerelle de management.
no shutdown
Active l'interface VLAN de gestion. Sur un switch L2, les ports physiques sont up par défaut, mais les interfaces VLAN (SVI) sont down par défaut — le no shutdown est donc nécessaire sur les SVIs.
ip default-gateway <IP_routeur>
Définit la passerelle par défaut du switch lui-même (pour que ses paquets de management puissent franchir un routeur). Ne concerne que le trafic généré par le switch, pas le trafic des hôtes.
Spanning Tree & vérification
spanning-tree portfast
Active PortFast sur un port access : le port passe immédiatement en état Forwarding, sans attendre les 30 secondes STP. Uniquement sur les ports vers des postes clients, jamais vers un autre switch (risque de boucle).
show spanning-tree vlan <id>
Affiche l'état STP par VLAN : quel switch est le Root Bridge, et l'état de chaque port (FWD = Forwarding, BLK = Blocking, LIS = Listening, LRN = Learning).
show mac address-table
Affiche la table CAM : correspondances MAC ↔ port ↔ VLAN. Utile pour tracer physiquement où est connecté un équipement spécifique dans le réseau.
show ip interface brief
Affiche un résumé de toutes les interfaces avec leur IP et leur état (up/down). Permet de vérifier rapidement que l'interface de gestion est bien active.

Switch Niveau 3
Couches 2 + 3 — Commutation + Routage inter-VLAN
Couches 2 + 3
Le switch L3 réalise tout ce que fait un switch L2, plus la capacité de router entre les VLANs grâce aux SVIs et à la commande ip routing. Il peut remplacer un routeur pour tout le trafic interne. Seules les commandes supplémentaires sont listées ici.
Routage inter-VLAN via SVIs
ip routing
Active le moteur de routage IP sur le switch L3. C'est LE point critique qui différencie un switch L2 d'un switch L3 en fonctionnement.

Sans cette commande, même avec des SVIs configurées et des IPs attribuées, aucun routage inter-VLAN ne se fera : les VLANs restent isolés comme sur un switch L2.

⚠️ Cette commande est exclusivement utile sur un switch L3. Sur un routeur, le routage IP est natif et toujours actif — ip routing n'a pas de raison d'y figurer.
interface vlan <id>
Crée une SVI (Switched Virtual Interface) pour un VLAN. Sur un switch L3, cette interface a un double rôle : passerelle par défaut de toutes les machines du VLAN, ET interface de routage entre VLANs.
ip address <IP> <masque>
Attribue l'IP à la SVI. Cette adresse sera configurée comme passerelle par défaut sur tous les postes du VLAN correspondant. Exemple : ip address 192.168.10.254 255.255.255.0 pour le VLAN 10.
no shutdown
Active la SVI. Les interfaces VLAN (SVIs) sont shutdown par défaut sur un switch L3 — même si l'IP est configurée, la SVI reste inactive jusqu'au no shutdown.

C'est un piège fréquent d'examen : on a fait ip routing, on a configuré l'IP de la SVI, mais on a oublié no shutdown. Résultat : le routage inter-VLAN ne fonctionne pas et show ip interface brief montre la SVI en down/down.
switchport trunk encapsulation dot1q
Définit le protocole d'encapsulation 802.1Q pour les trames taguées. Obligatoire sur les switches L3 Cisco avant de configurer un port en mode trunk. Sur les switches L2, 802.1Q est implicite et cette commande n'existe pas.
Lien routed vers le routeur (port L3)
no switchport
Transforme un port physique en port routed (couche 3). Le port sort du domaine de commutation et devient une interface IP standard. Utilisé pour créer le lien point-à-point /30 vers le routeur.
ip route 0.0.0.0 0.0.0.0 <IP_routeur>
Route par défaut : tout trafic sans route explicite est envoyé vers le routeur (qui assure la sortie vers Internet). N'a de sens qu'après ip routing — sans routage activé, cette commande ne peut pas être utilisée par le switch.
Vérification
show ip route
Affiche la table de routage du switch L3. Codes : C = connecté directement, S = statique, S* = route par défaut candidate. Si cette commande n'affiche rien d'utile, vérifier que ip routing est bien activé.
show ip interface brief
Résumé de toutes les interfaces (SVIs + ports routed) avec leur IP et état. La colonne d'état affiche up/up (fonctionnel), down/down (inactif, vérifier no shutdown) ou up/down (problème de protocole).

Routeur
Couche 3 — Interconnexion, NAT, DHCP, ACL
Couche 3
Le routeur gère la sortie vers l'extérieur (Internet, WAN), le NAT, le DHCP serveur, et le filtrage ACL. Il peut aussi assurer le routage inter-VLAN via des sous-interfaces (router-on-a-stick) quand il n'y a pas de switch L3. Attention : toutes ses interfaces physiques sont shutdown par défaut.
Interfaces physiques
interface fa0/0
Sélectionne l'interface physique FastEthernet. Sur un routeur, les interfaces sont nommées différemment selon le modèle : fa0/0, gi0/0, se0/0/0 (Serial).
no shutdown
Indispensable sur toutes les interfaces physiques d'un routeur. Contrairement aux switches où les ports sont up par défaut, les interfaces d'un routeur sont administrativement down par défaut.

Même parfaitement configurée (IP, masque, encapsulation), une interface routeur ne fonctionnera pas sans no shutdown. C'est le premier réflexe à avoir lors du diagnostic d'un routeur qui ne répond pas.
Router-on-a-stick (sans switch L3)
interface fa0/0.<id_vlan>
Crée une sous-interface pour le router-on-a-stick. La convention est d'utiliser le numéro du VLAN comme numéro de sous-interface (ex: fa0/0.101 pour le VLAN 101).
encapsulation dot1Q <id_vlan>
Associe la sous-interface au VLAN correspondant via l'encapsulation 802.1Q. Le routeur peut ainsi distinguer les trames de chaque VLAN arrivant sur le trunk.
Routage statique
ip route <réseau> <masque> <next-hop>
Route statique : "pour atteindre ce réseau, envoie les paquets vers cette IP". Dans un scénario avec switch L3, le routeur a besoin d'une route vers les réseaux internes (VLANs) via l'IP du switch L3.
ip route 0.0.0.0 0.0.0.0 <IP_FAI>
Route par défaut vers Internet : tout trafic sans route explicite part vers le FAI. Correspond au S* dans la table de routage.
show ip route
Affiche la table de routage. Codes : C = connecté, S = statique, R = RIP, O = OSPF, S* = route par défaut. La commande de vérification de référence.
NAT / PAT (accès Internet)
ip nat inside
Marque l'interface comme côté réseau privé (LAN). À appliquer sur l'interface connectée au réseau interne. Obligatoire avant toute règle NAT.
ip nat outside
Marque l'interface comme côté Internet. À appliquer sur l'interface connectée au FAI. Le routeur appliquera la traduction entre ces deux zones.
ip nat inside source list <ACL> interface <IF> overload
Active le PAT (NAT avec surcharge) : toutes les IPs privées sortent avec l'IP publique de l'interface, différenciées par les ports. C'est le fonctionnement classique de toute box Internet.
show ip nat translations
Affiche les entrées NAT actives : quelle IP:port interne correspond à quelle IP:port externe. Indispensable pour diagnostiquer les problèmes de NAT.
DHCP serveur
ip dhcp excluded-address <début> <fin>
Exclut une plage d'IPs de la distribution DHCP. À configurer avant la création du pool pour réserver les IPs fixes des équipements réseau (passerelle, serveurs, imprimantes...).
ip dhcp pool <nom>
Crée un pool DHCP nommé. Suivi des commandes network, default-router, dns-server et éventuellement lease pour la durée du bail.
network <réseau> <masque>
Définit la plage d'adresses à distribuer. Le routeur distribuera toutes les IPs de ce réseau sauf les exclues et lui-même.
default-router <IP_passerelle>
Indique aux clients la passerelle par défaut à utiliser. Généralement l'IP de la SVI du switch L3 ou de l'interface du routeur côté LAN.
ip helper-address <IP_serveur_DHCP>
Sur une interface, redirige les broadcasts DHCP vers un serveur DHCP centralisé. Permet à un seul serveur DHCP de servir plusieurs VLANs à travers le routeur (relay DHCP).

PC / Hôte Non (hors Cisco CLI) —
Switch L2 Non Ne route pas, c'est normal

Mémo rapide pour l'examen :
ip routing → uniquement sur switch L3, c'est l'interrupteur qui active le routage.
no shutdown sur SVIs → toujours, que ce soit sur switch L2 (gestion) ou L3 (routage).
no shutdown sur ports physiques → uniquement sur le routeur (les switches mettent leurs ports up tout seuls).

⚠️ Scénario piège classique d'examen

Un switch L3 est configuré : les VLANs existent, les SVIs ont leurs IPs, ip routing est activé. Pourtant les hôtes de VLANs différents ne se pingent pas.

Cause probable : oubli du no shutdown sur les SVIs. Vérification : show ip interface brief → les SVIs apparaissent en down/down.

Deuxième cause fréquente : sur un routeur, l'interface physique est bien configurée mais pas de no shutdown → elle reste en administratively down. La sous-interface router-on-a-stick ne peut jamais fonctionner si son interface parente est down.

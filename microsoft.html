🟢 Niveau 1 — Fondamentaux Windows Server

Création de l'Active Directory (AD DS)

Promotion d'un serveur en contrôleur de domaine

1. Installation du rôle AD DS

  1. 1Gestionnaire de serveur → Gérer → Ajouter des rôles et fonctionnalités
  2. 2Cliquer Suivant jusqu'à « Rôles de serveur »
  3. 3Cocher Services AD DS → valider l'ajout de fonctionnalité
  4. 4Cliquer Suivant plusieurs fois → Installer → Fermer

2. Promotion en contrôleur de domaine

  1. 1Cliquer sur le triangle jaune (notification) dans le gestionnaire de serveur
  2. 2Cliquer sur « Promouvoir ce serveur en contrôleur de domaine »
  3. 3Sélectionner « Ajouter une nouvelle forêt »
  4. 4Nom de domaine racine : tssr.local
  5. 5Attendre quelques secondes (chargement), puis saisir le mot de passe DSRM : Reponse2025 (deux fois)
  6. 6Cliquer Suivant plusieurs fois — le nom NetBIOS (TSSR) apparaît automatiquement
  7. 7Vérification de la configuration → Installer
  8. 8Cliquer sur Fermer → le serveur redémarre automatiquement
⚠️ Le serveur va redémarrer à l'issue de la promotion. C'est normal.

Routage NAT

Activer le NAT via le rôle Accès à distance

1. Installation du rôle

  1. 1Gestionnaire de serveur → Gérer → Ajouter des rôles
  2. 2Cliquer Suivant jusqu'à la sélection des rôles
  3. 3Cocher Accès à distance
  4. 4Dans « Services de rôle », cocher Routage (DirectAccess et VPN se cochent automatiquement)
  5. 5Suivant → Suivant → Installer

2. Configuration du NAT

  1. 1Gestionnaire de serveur → Outils → Routage et accès distant
  2. 2Clic droit sur le serveur → Configurer et activer le routage
  3. 3Cliquer Suivant jusqu'à la fenêtre de configuration
  4. 4Sélectionner NAT (Network address translation)
  5. 5Sélectionner la carte WAN → Suivant → Terminer
💡 Si une boîte d'erreur apparaît, choisir « Continuer l'exécution et ignorer les erreurs » et cliquer OK. Retourner ensuite dans Routage et accès distant pour vérifier les propriétés du WAN.
⚠️ Vérifier que l'antivirus est désactivé (serveur et client) et que le proxy est désactivé (serveur et client).

Réservation DHCP

Installation et configuration d'une étendue avec réservation d'adresse IP

1. Installation du rôle DHCP

  1. 1Gestionnaire de serveur → Gérer → Ajouter des rôles
  2. 2Cocher Serveur DHCP → Suivant → Installer
  3. 3Cliquer sur le triangle jaune → Terminer la configuration DHCP
  4. 4Suivant → ne pas changer l'autorisation (laisser le compte Admin courant) → Valider → Fermer

2. Création d'une étendue

  1. 1Outils → Gestionnaire DHCP → clic droit sur IPv4 → Nouvelle étendue
  2. 2Nommer l'étendue (ex : LAN-TP) et décrire
  3. 3Plage IP de début à fin (ex : 192.168.102.100 à 192.168.102.199)
  4. 4Ajouter les exclusions (ex : 192.168.102.100 pour le serveur fixe)
  5. 5Durée du bail : généralement 8 heures (4 jours pour fixes)
  6. 6Configurer maintenant les options : passerelle, DNS, domaine
  7. 7WINS → Suivant sans rien renseigner
  8. 8Activer l'étendue maintenant → Terminer

3. Réservation d'adresse IP (par adresse MAC)

  1. 1Sur le client, récupérer l'adresse MAC avec :
cmd — client
ipconfig /all
Relever l'adresse physique (ex : 08-00-27-56-FA-E2)
  1. 2DHCP → IPv4 → clic droit Réservation → Nouvelle réservation
  2. 3Nom, adresse IP souhaitée, adresse MAC (sans tirets), description → Ajouter
✅ Le client obtiendra toujours la même IP à chaque connexion.
🔵 Niveau 2 — Services réseau

DNS — Création d'une zone et ajout d'hôtes

Configuration du serveur DNS principal sur Windows Server

1. Propriétés du serveur DNS

  1. 1Gestionnaire de serveur → clic droit sur DNS → Gestionnaire DNS
  2. 2Clic droit sur le serveur DNS (ex : DNS1) → Propriétés
  3. 3Onglet Interfaces : décocher IPv6, cocher « Uniquement les adresses IP suivantes » (conserver l'IP statique du serveur)
  4. 4Onglet Redirecteurs → Modifier → ajouter 8.8.8.8 → OK
  5. 5Onglet Avancé → rien à changer → OK

2. Création d'une zone de recherche directe

  1. 1Zone de recherche directe → clic droit → Nouvelle zone → Suivant
  2. 2Type : Zone principale → Suivant
  3. 3Nom de la zone : labo.tssr → Suivant
  4. 4Créer un nouveau fichier de zone → Suivant
  5. 5Mises à jour dynamiques : Ne pas autoriser → Suivant → Terminer
📌 La zone créée contient automatiquement deux enregistrements : SOA (Start of Authority) et NS (Name Server).

3. Zone de recherche inversée

  1. 1Zone de recherche inversée → clic droit → Nouvelle zone
  2. 2Zone principale → Zone de recherche inversée IPv4
  3. 3ID réseau : saisir les 3 premiers octets (ex : 192.168.240)
  4. 4Suivant → Suivant → Terminer

4. Ajout d'hôtes (enregistrements A)

  1. 1Clic droit dans la zone labo.tssr → Nouvel hôte (A ou AAAA)…
  2. 2Nom : ex Routeur, Adresse IP : 192.168.240.254
  3. 3Cocher Créer un pointeur PTR (visible dans zone inversée)
  4. 4Cliquer Ajouter un hôte → OK

5. Alias CNAME (raccourci de nom)

  1. 1Clic droit → Nouvel alias (CNAME)…
  2. 2Nom de l'alias : ex rt
  3. 3Parcourir jusqu'à l'hôte cible (ex : Routeur.labo.tssr) → OK
cmd — test ping DNS
ping routeur.labo.tssr
Vérifie la résolution de nom DNS vers l'adresse IP

DNS Secondaire (zone secondaire)

Partage de la zone DNS entre deux serveurs pour la redondance

Sur le serveur secondaire (DNS2)

  1. 1Installer le rôle DNS (idem que le principal)
  2. 2Gestionnaire DNS → Zone de recherche directe → Nouvelle zone
  3. 3Type : Zone secondaire
  4. 4Nom de zone : labo.tssr → Suivant
  5. 5Serveurs maître : saisir l'IP du DNS1 (ex : 192.168.240.211) → Suivant → Terminer

Sur le serveur principal (DNS1)

  1. 1Ajouter un nouvel hôte A pour DNS2 (ex : 192.168.240.212)
  2. 2Sur DNS2 → clic droit sur la zone → Charger à nouveau (ou rafraîchir)
✅ Sur DNS2, la zone est en lecture seule (zone secondaire). Seul DNS1 peut créer/modifier des enregistrements.

Profils itinérants

Stocker les profils utilisateurs sur le serveur (accessible de n'importe quel poste)

1. Partage du dossier Profil

  1. 1Créer un dossier sur le serveur (ex : D:\Profil)
  2. 2Clic droit → Propriétés → onglet Partage → Partage avancé
  3. 3Cocher « Partager ce dossier », nom du partage : Profil
  4. 4Autorisations → cocher Contrôle total pour « Tout le monde » → Appliquer → OK → Fermer

2. Configurer le profil dans l'AD

  1. 1Utilisateurs et ordinateurs AD → double-cliquer sur l'utilisateur
  2. 2Onglet Profil → Chemin du profil :
Chemin du profil utilisateur
\\AD01\profil\p.sorin
Remplacer p.sorin par le login de l'utilisateur

3. Lecteur réseau (optionnel)

  1. 1Poste de travail → Ordinateur → Connecter un lecteur réseau
  2. 2Lettre : S:, Dossier : \\srv00\profil
  3. 3Cocher « Se reconnecter lors de la connexion » → Terminer

Serveur FTP (IIS)

Créer un serveur FTP avec accès isolé par utilisateur AD

1. Prérequis — Créer les utilisateurs dans l'AD

  1. 1Dans l'AD, créer une OU (ex : @greta)
  2. 2Créer 2 utilisateurs de test (ex : Alex Terrieur et Samantha LOT)

2. Installation du rôle IIS + FTP

  1. 1Gestionnaire de serveur → Gérer → Ajouter des rôles
  2. 2Sélectionner Serveur Web (IIS)
  3. 3Dans les services de rôle, cocher Serveur FTP et Extensibilité FTP
  4. 4Suivant → Installer

3. Arborescence des dossiers

  1. 1Aller dans C:\inetpub\ftproot
  2. 2Créer un dossier TSSR
  3. 3Dans TSSR, créer un sous-dossier par utilisateur (ex : aterrieur, slot)
  4. 4Créer un fichier test dans chaque dossier
  5. 5Propriétés de chaque dossier → onglet Sécurité → Modifier → Ajouter l'utilisateur correspondant

4. Configuration IIS (site FTP)

  1. 1Gestionnaire IIS → Sites → clic droit → Ajouter un site FTP
  2. 2Nom : FTP AD, Chemin : C:\inetpub\ftproot
  3. 3Liaison : port 21, Pas de SSL
  4. 4Authentification : cocher De base
  5. 5Autorisation : Tous les utilisateurs, Autorisations : Lecture et Écriture
  6. 6Cliquer Terminer

5. Test avec WinSCP

  1. 1Lancer WinSCP, protocole : FTP
  2. 2Hôte : IP du serveur (ex : 192.168.240.201), port 21
  3. 3Nom d'utilisateur : login AD (ex : slot), Mot de passe : celui de l'AD
  4. 4Se connecter → on arrive dans le dossier de l'utilisateur et on voit son fichier
🟡 Niveau 3 — AGDLP

Concept AGDLP

Méthode d'organisation des droits d'accès dans Active Directory
AGDLP = Account → Global → Domain Local → Permission
Principe : on n'affecte jamais un utilisateur directement à un dossier. On passe par des groupes.
SigleSignificationRôle
A Account (Compte utilisateur) L'utilisateur AD
G Global group Groupe global par département/métier (ex : g_rh, g_informatique)
DL Domain Local group Groupe local portant les droits sur une ressource (ex : dl_dossier_RH_C)
P Permission F=Full, C=Change (Modifier), R=Read (Lecture)
Règle AGDLP : A → G → DL → P
Un utilisateur est mis dans un G, le G est mis dans un DL, le DL reçoit les droits sur le dossier.

Portée des groupes

TypeVisibilité
Domaine local (DL)Visible uniquement dans le domaine où il a été créé
Global (G)Visible par tous les domaines (parent et enfant)
UniverselleVisible sur tous les domaines de toute la forêt

Partage & Droits AGDLP — Étape par étape

Procédure complète de mise en place AGDLP sur un dossier partagé

1. Partage du dossier parent

  1. 1Clic droit sur le dossier de partage → Propriétés → onglet Partage
  2. 2Partage avancé → cocher « Partager ce dossier »
  3. 3Autorisations → Tout le monde : Contrôle total → Appliquer → OK → Fermer

2. Création des sous-dossiers et des groupes DL

  1. 1Créer les sous-dossiers dans le dossier partagé (ex : RH, Support)
  2. 2Dans l'AD, dans l'OU Groupe, créer 3 groupes DL par dossier :
Nom du groupeTypePortéePermission
dl_dossier_RH_FSécuritéDomaine localFull (Contrôle total)
dl_dossier_RH_CSécuritéDomaine localChange (Modification)
dl_dossier_RH_RSécuritéDomaine localRead (Lecture)

3. Désactivation de l'héritage sur les sous-dossiers

  1. 1Clic droit sur RH → Propriétés → onglet Sécurité → Avancé
  2. 2Cliquer Désactiver l'héritage → Convertir les autorisations héritées en autorisations explicites
  3. 3OK → retour Sécurité → Modifier
  4. 4Supprimer le groupe Utilisateurs
  5. 5Ajouter les 3 groupes DL du dossier RH (taper dl → Vérifier les noms → sélectionner les 3)
  6. 6Attribuer les droits manuellement : F → Contrôle total / C → Modifier / R → Lecture+Exécution
  7. 7Appliquer → OK
🔁 Répéter l'opération pour chaque sous-dossier (Support, Compta, etc.)

4. Association des groupes G aux groupes DL

Exemple de demande :

Groupe G (métier)Groupe DL (ressource)Résultat
g_rhdl_dossier_RH_CLe service RH peut modifier le dossier RH
g_directiondl_dossier_RH_RLa direction peut lire le dossier RH
g_supportdl_dossier_Support_CLe support peut modifier le dossier Support
  1. 1Double-cliquer sur dl_dossier_RH_C → onglet Membres → Ajouter
  2. 2Taper g_rh → Vérifier les noms → OK
  3. 3Répéter pour chaque association DL ↔ G

Script agdlp.bat

Automatisation de la création de dossier + groupes DL + droits NTFS
script_agdlp.bat
@echo off set domaine=labo set extension=tssr REM ETAPE01 - Demander le nom du dossier set /p dossier=Veuillez saisir le nom du dossier : mkdir e:\partage\%dossier% REM ETAPE02 - Créer les 3 groupes DL (F/C/R) dsadd group cn=dl_Dossier_%dossier%_F,ou=groupe,ou=@%domaine%,dc=%domaine%,dc=%extension% -scope l dsadd group cn=dl_Dossier_%dossier%_C,ou=groupe,ou=@%domaine%,dc=%domaine%,dc=%extension% -scope l dsadd group cn=dl_Dossier_%dossier%_R,ou=groupe,ou=@%domaine%,dc=%domaine%,dc=%extension% -scope l REM ETAPE03 - Désactiver l'héritage icacls e:\partage\%dossier% /inheritance:d REM ETAPE04 - Supprimer le groupe Utilisateurs icacls e:\partage\%dossier% /remove:g "Utilisateurs" REM ETAPE05 - Ajouter les 3 groupes DL avec leurs droits icacls e:\partage\%dossier% /grant dl_Dossier_%dossier%_F:(OI)(CI)(F) icacls e:\partage\%dossier% /grant dl_Dossier_%dossier%_C:(OI)(CI)(M) icacls e:\partage\%dossier% /grant dl_Dossier_%dossier%_R:(OI)(CI)(RX) pause
Lancer ce script autant de fois qu'il y a de dossiers à créer. Adapter le chemin e:\partage selon votre configuration.
⚠️ Adapter les variables domaine et extension à votre domaine AD (ex : tssr / local).

Mappage de lecteurs réseau

3 méthodes pour connecter un dossier partagé comme lecteur réseau

Méthode 1 — Navigation directe

Barre d'adresse de l'Explorateur
\\AD01\partage
Affiche tous les sous-dossiers partagés. L'utilisateur ne voit que ceux auxquels il a accès.

Méthode 2 — Lecteur réseau via l'interface

  1. 1Poste de travail → Ordinateur → Connecter un lecteur réseau
  2. 2Lettre : P:, Dossier : \\AD01\partage\Production
  3. 3Cocher « Se reconnecter lors de la connexion » → Terminer

Méthode 3 — Script .bat (mappage automatique)

mappage.bat
@echo off net use p: \\ad01\partage\production net use t: \\ad01\partage\commun net use z: \\ad01\partage\direction
Crée 3 lecteurs réseau simultanément. Idéal pour automatiser via GPO (script d'ouverture de session).
✅ Pour automatiser pour tous les utilisateurs d'un groupe → utiliser les GPO (Préférences utilisateur → Mappages de lecteurs).
🔴 Niveau 4 — Rôles FSMO

Les 5 rôles FSMO

Flexible Single Master Operations — rôles uniques dans la forêt/domaine AD
Pourquoi les FSMO ? Active Directory est multi-maître (tous les DC peuvent écrire), mais certaines opérations doivent être gérées par un seul DC pour éviter les conflits. Ces 5 rôles garantissent cohérence et unicité.
RôlePortéeRôle principalImportance
Contrôleur de schéma
(Schema Master)		 Forêt Gère la structure de la base AD (types d'objets, attributs). Nécessaire pour modifier le schéma (ex : installer Exchange). Très forte
Maître des noms de domaine
(Domain Naming Master)		 Forêt Autorise ou refuse l'ajout/suppression de domaines dans la forêt. Élevée (rare)
Émulateur PDC
(PDC Emulator)		 Domaine Horloge officielle du domaine, gère les mots de passe récents, édite les GPO, compatibilité NT4. Très forte ★
Maître RID
(RID Master)		 Domaine Distribue des blocs de RID aux DC pour créer des SID uniques pour chaque nouvel objet AD. Élevée
Maître d'infrastructure
(Infrastructure Master)		 Domaine Met à jour les références inter-domaines (objets d'un autre domaine). Quasi inutile dans un domaine unique. Variable

Vérifier les rôles FSMO

cmd — sur n'importe quel DC
netdom query fsmo
Affiche les 5 rôles FSMO et le DC qui les détient actuellement

Transfert de rôle FSMO (Transfer)

Déplacer un rôle d'un DC vers un autre — les deux DC sont opérationnels
Transfer = les deux serveurs sont en ligne. On déplace proprement le rôle.
Exemple : transférer le rôle RID Master de AD01 vers AD02.

Procédure de transfert avec ntdsutil

cmd — sur AD02 (serveur cible)
ntdsutil roles connections connect to server ad02 quit transfer RID master
1. Lancer ntdsutil → 2. Entrer en mode "roles" → 3. Se connecter au serveur cible → 4. Quitter connections → 5. Transférer le rôle
  1. 1Lancer ntdsutil en ligne de commande
  2. 2Taper roles → on entre en mode fsmo maintenance
  3. 3Taper connections → on entre en mode server connections
  4. 4Taper connect to server ad02 → connexion établie
  5. 5Taper quit → retour en mode fsmo maintenance
  6. 6Taper transfer RID master → boîte de dialogue de confirmation → cliquer Oui
  7. 7Taper quit puis quit pour sortir complètement
  8. 8Vérifier : netdom query fsmo
Commande ntdsutilAction
transfer schema masterTransfert du rôle Contrôleur de schéma
transfer naming masterTransfert du rôle Maître des noms de domaine
transfer PDCTransfert du rôle Émulateur PDC
transfer RID masterTransfert du rôle Maître RID
transfer infrastructure masterTransfert du rôle Maître d'infrastructure

Seizing (récupération d'urgence)

Récupérer un rôle FSMO quand le DC qui le détient est hors service
🚨 Seize = le DC maître est éteint ou en panne. On force la prise du rôle sur un autre DC. À n'utiliser qu'en dernier recours — le DC d'origine NE DOIT JAMAIS être remis en ligne par la suite.

Procédure de seizing avec ntdsutil

cmd — sur AD01 (DC de récupération)
ntdsutil roles connections connect to server ad01 quit seize RID master
On se connecte sur son propre serveur (celui qui récupère le rôle), puis on force la saisie.
  1. 1Lancer ntdsutil sur le DC de récupération (AD01)
  2. 2Taper roles → mode fsmo maintenance
  3. 3Taper connections → connect to server ad01 → quit
  4. 4Taper seize RID master → boîte de confirmation → Oui
  5. 5Message d'erreur LDAP attendu (le DC d'origine est HS) → le seizing continue quand même
  6. 6Taper quit puis quit
  7. 7Vérifier : netdom query fsmo → le rôle RID est sur AD01

NIV. 1 Fondamentaux Profil itinérant · DHCP de base
Profil itinérant — Configuration AD · Partage
01
Créer le dossier profil sur le second disque dur (ex. S:\profil)
ℹCe dossier contiendra les profils de chaque utilisateur. Il doit être sur une partition dédiée au partage, pas sur C:.
02
Clic droit sur le dossier → Propriétés → onglet Partage → Partage avancé
03
Cocher Partager ce dossier, nommer le partage profil
04
Cliquer sur Autorisations → sélectionner Tout le monde → cocher Contrôle total → OK
⚠Le contrôle total ici est côté partage SMB. Les droits NTFS (onglet Sécurité) peuvent être affinés séparément pour plus de granularité.
05
Dans Utilisateurs et ordinateurs Active Directory, sélectionner les utilisateurs concernés (ex. groupe Formation) → clic droit → Propriétés
06
Onglet Profil → champ Chemin du profil → entrer le chemin UNC avec variable :
\\192.168.100.240\profil\%username%
✓%username% est automatiquement remplacé par le nom de connexion de chaque utilisateur → chaque user obtient son propre sous-dossier.
07
Cliquer Appliquer puis OK. Vérifier sur un utilisateur individuel : onglet Profil doit afficher le chemin réseau renseigné.
08
Se connecter depuis un PC client avec l'un des utilisateurs du groupe pour valider la création automatique du profil réseau.

NIV. 2 Intermédiaire GPO avancées · AGDLP · Mappage de lecteurs
Gestion des fichiers — AGDLP + GPO Mappage AGDLP · Script · Mappage
01
Script de création AGDLP — Exécuter le script batch (adapter le nom de domaine en haut du script) et saisir le nom du dossier à créer :
REM ETAPE01 — Créer le dossier mkdir s:\%dossier% REM ETAPE02 — Créer les 3 groupes DL (F/C/R) dsadd group cn=dl_Dossier_%dossier%_F,ou=groupes,... dsadd group cn=dl_Dossier_%dossier%_C,ou=groupes,... dsadd group cn=dl_Dossier_%dossier%_R,ou=groupes,... REM ETAPE03 — Désactiver l'héritage NTFS icacls s:\%dossier% /inheritance:d REM ETAPE04 — Supprimer le groupe Utilisateurs icacls s:\%dossier% /remove:g "Utilisateurs" REM ETAPE05 — Appliquer droits sur les 3 groupes DL icacls s:\%dossier% /grant dl_Dossier_%dossier%_F:(OI)(CI)(F) icacls s:\%dossier% /grant dl_Dossier_%dossier%_C:(OI)(CI)(M) icacls s:\%dossier% /grant dl_Dossier_%dossier%_R:(OI)(CI)(RX)
⚠Penser à modifier le nom de domaine en haut du script avant exécution. Répéter pour chaque dossier (Informatique, Formation, Administration, Public).
02
Après création par script, appliquer les droits de partage manuellement : Clic droit dossier → Propriétés → Sécurité → Partage avancé → Autorisations → Tout le monde → Modifier + Lecture (pas besoin de Contrôle total)
03
Ajouter les membres dans les groupes DL — Dans l'AD, naviguer vers chaque groupe DL et ajouter les groupes globaux correspondants :
dl_Dossier_public_F ← ajouter : Utilisateurs du domaine dl_Dossier_informatique_F ← ajouter : g_informatique dl_Dossier_formation_F ← ajouter : g_formateur dl_Dossier_administration_F ← ajouter : g_greta, g_delegue
04
GPO — Mappage Public par script — Créer le script mappage_public.bat :
net use p: \\192.168.100.240\public
05
Gestionnaire de serveur → Outils → Gestion des stratégies de groupe → clic droit sur le domaine @greta → Créer un objet GPO dans ce domaine et le lier ici…
Nom : Mappage_public_script_P
06
Clic droit sur la GPO → Modifier → Configuration utilisateur → Stratégies → Paramètres Windows → Scripts (Ouverture de session) → double-clic sur Ouverture de session
07
Cliquer Ajouter → Parcourir → copier le fichier mappage_public.bat dans le dossier Logon de la GPO → le sélectionner → OK → OK
✓Utiliser "Afficher les fichiers" pour coller le .bat dans le dossier Logon accessible depuis la fenêtre Parcourir.
08
GPO — Mappage par Préférences (sans script) — Pour les OU Informatique et Formation : GPO → Modifier → Configuration utilisateur → Préférences → Paramètres Windows → Mappages de lecteurs → Nouveau → Lecteur mappé
Informatique → \\192.168.100.240\informatique Lettre: I Formation → \\192.168.100.240\formation Lettre: F Administration → \\192.168.100.240\administration Lettre: G
ℹPour Administration (délégués) : créer la GPO au niveau du domaine @greta et faire un filtrage de sécurité pour cibler uniquement g_greta et g_delegue. Retirer "Utilisateurs authentifiés" du filtrage et les remettre dans l'onglet Délégation en Lecture seule.
09
Appliquer les stratégies sur serveur et client :
gpupdate /force
Redémarrer le PC client, se reconnecter avec un utilisateur et vérifier les lecteurs réseau dans le Poste de travail.

GPO — Interdire le Bloc-notes GPO · Restriction · Système
01
Gestion des stratégies de groupe → clic droit sur @greta → Créer un objet GPO…
Nom : BLOC-NOTEPAD
02
Clic droit GPO → Modifier → Configuration utilisateur → Stratégies → Modèles d'administration → Système → double-clic sur Ne pas exécuter les applications Windows spécifiées
03
Sélectionner Activé → cliquer sur Afficher… → ajouter dans la liste :
NOTEPAD.EXE
→ OK → Appliquer → OK
ℹCette méthode (Modèles d'administration → Système) est plus fiable que la méthode par "Stratégies de restriction logicielle" sur les versions récentes de Windows 10/11.
04
Filtrage de sécurité — Onglet Étendue → retirer Utilisateurs authentifiés → ajouter le groupe ciblé (ex. g_greta)
✓Onglet Délégation → Ajouter → "Utilisateurs authentifiés" → autorisation : Lecture. Sans cela la GPO ne peut pas être lue du tout.
05
Appliquer sur serveur et client :
gpupdate /force
Redémarrer le client → se connecter avec un utilisateur du groupe g_greta → tenter d'ouvrir le Bloc-notes → message de restriction attendu.
NIV. 3 Avancé Déploiement MSI · DNS inverse · CNAME
Déploiement MSI Firefox par GPO GPO · MSI · Installation logiciel
01
Télécharger Firefox au format .msi (pas .exe) depuis :
https://www.mozilla.org/fr/firefox/all/#product-desktop-release
02
Créer le dossier S:\Appli et y déposer le fichier Firefox.msi
03
Rendre le partage invisible (nom avec $) : Propriétés → Partage avancé → Ajouter un nom de partage
Nom du partage : appli$ Description : INVISIBLE
Supprimer le partage initial "Appli" → ne garder que appli$ → OK
ℹUn partage terminé par $ est invisible dans l'explorateur réseau mais accessible directement via le chemin UNC.
04
Dans l'AD : créer une OU PC → y glisser le compte ordinateur PC client (depuis le conteneur Computers)
05
Gestion des stratégies de groupe → clic droit sur OU PC → Créer un objet GPO…
Nom : INSTALL-MSI
06
Clic droit GPO → Modifier → Configuration ordinateur → Stratégies → Paramètres du logiciel → Installation de logiciel → clic droit à droite → Nouveau → Package…
07
Dans la barre d'adresse, saisir le chemin UNC du partage caché :
\\192.168.100.240\appli$
Sélectionner Firefox.msi → Ouvrir → Mode de déploiement : Attribué → OK
⚠Ne jamais utiliser un chemin local (C:\…) dans une GPO d'installation logiciel. Le chemin doit impérativement être UNC (\\serveur\partage) pour que les clients puissent y accéder.
08
Actualiser (icône verte) → Firefox apparaît avec statut "Attribué". Fermer les fenêtres GPO.
09
Sur le serveur puis le client :
gpupdate /force
Sur le client : valider le redémarrage demandé → reconnecter → Firefox est installé automatiquement au démarrage.

Installation du Contrôleur de domaine secondaire (PowerShell) ADDS · PowerShell · DC · DNS secondaire
01

02

Import-Module ADDSDeployment
03

Install-ADDSDomainController -InstallDNS -Credential (Get-Credential "greta.eval\administrateur") -DomainName "greta.eval"
ℹUne fenêtre d'authentification Windows s'ouvre → entrer le mot de passe administrateur du domaine. Puis PowerShell demande le mot de passe SafeMode (DSRM).
04
Renseigner le mot de passe DSRM :
SafeModeAdministratorPassword : *********** Confirmer SafeModeAdministratorPassword : ********
05
Confirmer l'opération → taper O (Oui) → la promotion s'exécute, le serveur redémarre automatiquement (attendre ~2 minutes)
⚠Le serveur reste longtemps sur l'écran "Application des paramètres" au redémarrage — c'est normal pour la réplication ADDS.

✓Après rafraîchissement, AD02.greta.eval doit apparaître dans la liste Serveurs de noms (NS) de la zone greta.eval.
